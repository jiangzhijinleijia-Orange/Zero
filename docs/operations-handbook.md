# 蒼理塾 学習管理アプリ — 運営メンバー・ハンドブック

> この一枚は、運営メンバーが **「自信を持って人に勧め、迷わず日々まわせる」** 状態になるためのものです。
> 技術の細部は別の文書([使い方ガイド](user-guide.md)・[デプロイ運用ガイド](deployment.md))に譲り、ここには
> **「なぜこうなっているか」と「あなたが何をすればいいか」** だけを書きます。読むのに5分、でも必要なことは全部入っています。

---

## 1. このアプリは何のためにあるか

蒼理塾は、沖縄を拠点とする無償の学習支援コミュニティです。日々の活動は Discord(琉球サーティー)の上で回ってきました。会話の基盤として Discord は十分に機能しています——けれど、ひとつだけ苦手なことがあります。**「状態」を持ち続けること**です。

誰の課題がどこまで進んだか。先週の模試は何点だったか。次の締切はいつか。こうした"動かない事実"を、チャットという"流れていく川"の中に置くと、すぐに過去ログの底へ沈みます。結果として——進捗は DM で個別に聞いて回るしかなく、成績は記憶とスクショに散らばり、せっかくのデータも「次に何を学ぶべきか」の判断にはつながらない。

このアプリは、その **"流れて消える状態"だけ** をすくい取り、残し、一覧で見えるようにする道具です。Discord を置き換えるものではありません。役割をはっきり分けます——

> **会話は Discord、状態はアプリ。**

この一行が、プロジェクト全体を貫く背骨です。

---

## 2. 三つの道具と、あなたの役割

アプリの裏側では、三つの無料サービスが手を繋いで動いています。仕組みを暗記する必要はありませんが、誰かに「これどうなってるの?」と聞かれたとき、一言で答えられると安心です。

| 道具 | たとえると | 役割 |
|---|---|---|
| **Vercel** | 校舎そのもの | アプリの画面をインターネットに届ける「場所」。みんなが開く URL はここ |
| **Supabase** | 鍵付きの書庫＋受付 | 成績・課題・予定の**保管庫**と、**ログインの本人確認**。「他人の成績は見えない」を守っているのもここ |
| **Discord** | 生徒証＋連絡網 | **ログインの身分証**。加えて招待リンクの配布と、課題が出たときの通知 |

三つに分けたのは、それぞれが「一つのことを得意とし、この規模なら無料」だからです。とりわけ、**機微な成績は Supabase の書庫に厳重に鍵をかけてしまい、Vercel は画面を見せるだけ**——という役割分担そのものが、安全設計になっています。

そして肝心の、**あなた(運営)の役割**。覚えておいてほしいのは、**アプリは基本的に、ひとりでに回る**ということです。サーバーの番をする必要も、お金の管理をする必要もありません。運営の仕事を突き詰めると、たった二つ——**入口を管理すること**(招待リンクを発行して配る)と、**つまずいた人に手を貸すこと**。それ以外は、アプリと開発担当が引き受けます。

---

## 3. 迷ったら立ち返る、三つの約束

機能の要望や運用の判断で迷ったとき、ここに照らせば道を間違えません。

**① 会話は Discord、状態はアプリ。**
チャット・雑談・お知らせ配信は Discord の領分です。「これ、アプリに付けたら便利では?」と思いついたとき、その多くは Discord でやれば済みます。アプリに持ち込むのは"状態"だけ——課題・成績・予定。この線引きが、アプリが太りすぎて誰にも使われなくなるのを防ぎます。

**② 生徒の手間は、限界まで減らす。**
生徒が「これは記録のための作業だ」と感じた瞬間に、アプリは使われなくなります。だから進捗の更新はボタン一つ(未着手 → 進行中 → 完了)。何かを足したくなったら、まず「導線を減らせないか」を先に考えてください。**足し算より引き算**です。

**③ 成績は、機微な情報として扱う。**
自分の成績を見られるのは、本人・担当の講師・運営だけ。他の生徒からは決して見えません。これはマナーやお願いではなく、データベースのいちばん深いところで**強制**されている仕組みです。仮にアプリに不具合が出ても、他人の成績が漏れることはない——その前提で、安心して生徒や保護者に説明してください。

## 4. 仲間を迎える、送り出す

このアプリは**招待制**です。Discord アカウントさえあれば誰でもログインボタンは押せますが、**招待されていない人には何も見えません**(画面が真っ白になる)。鍵をかけているのは「招待」、Discord はあくまで本人確認——この二重構造が入口を守っています。

| 場面 | やること |
|---|---|
| **迎える(招待)** | `/admin` で招待リンクを発行 → Discord の DM で本人へ。生徒なら発行時に**担当講師まで指定**できる。リンクは**72時間・1回限り**有効 |
| **登録(本人の操作)** | リンクを開く → **Discord でワンタップ・ログイン** → 表示名を入れて完了。新しいパスワードは不要 |
| **担当を変える** | `/admin` の「担当関係」で設定・終了。終了すると、その講師からはその生徒のデータが**即座に見えなくなる** |
| **送り出す(退会)** | **無効化**(ログインはできても何も見えなくなる)/ **完全削除**(Supabase でユーザーを消すと、本人の課題・成績も連動して消える) |

**未成年の生徒を招待する前に、保護者の同意を取ってください。** 成績という機微な情報を預かる以上、口頭ではなく**記録の残る形**(Google Forms など)で。ひな形は [consent-form.md](consent-form.md) にあります。順番は必ず——**同意 → 招待**。

※ 現在の本番 URL は `https://zero-drab.vercel.app` です。アプリを開くときは常にこの URL を使ってください。

## 5. 日々の運用リズム

実際のところ、運営は**ほとんど「何もしない」**のが正常です。アプリは放っておいても回ります。

| 頻度 | 誰が | やること |
|---|---|---|
| 新メンバーが入るとき | 運営 | 招待リンクを発行して Discord で送る |
| 毎週 | 講師 | 課題を作って担当生徒に割り当てる(Discord に通知が飛ぶ) |
| 日々 | 生徒 | 「今日やること」を見て、終わったら1タップで更新 |
| テスト後 | 生徒・講師 | 成績を登録する |
| ときどき | 運営 | 困っている人を助ける/退会者を無効化する |
| 月1回 | 運営 | KPI を確認する(§7) |

立ち上げの順番だけ、ひとつコツがあります。**いきなり生徒を入れず、まず身内で試す**こと。あなた＋運営数名＋講師1名で「課題を作る→生徒役で更新する→成績を入れる」を一周し、操作感を確かめてから、生徒数名のクローズドβへ進みます。

## 6. お金:絶対に課金させない仕組み

このアプリは、すべて無料枠の中で動いています。しかも「使いすぎたら課金」ではなく、**「使いすぎたら一時的に止まるだけ(課金されない)」サービスだけ**で組んであります。守る約束は、たった二つ:

> **・Supabase を Pro にアップグレードしない**
> **・Vercel にクレジットカードを登録しない**

この二つを守る限り、請求は**構造的に**発生しません。規模の目安として、数十人が数年使ってもデータ量は無料枠に余裕で収まります。新しい運営メンバーにも、この二つの約束だけは必ず引き継いでください。

---

## 7. うまくいっているかの、見方

数字を追う前に、**たった一つの観察ポイント**を共有します:

> **生徒が、言われなくても進捗を更新するか。**

これが起きていれば、このアプリは成功です。起きていなければ——機能を足すのではなく、**「もっと押しやすく」削る方向**で直します(その調整は開発担当がやります)。アプリの価値は機能の数ではなく、生徒が自然に触れ続けることにあります。

月に一度、補助的に KPI を確認します。[deployment.md](deployment.md#3-日常運用) の SQL をコピーして Supabase に貼るだけです。

| 指標 | 目安 |
|---|---|
| 週次アクティブ生徒率 | 70%以上 |
| 課題ステータス更新率 | 80%以上 |
| 成績登録の継続率 | 90%以上 |

これらは**ノルマではなく、機能の足し引きを判断する材料**です。数字が低いところは、生徒を責めるのではなく「導線のどこが重いか」を疑ってください。

---

## 8. 困ったとき、変えたいとき

画面がおかしい・エラーが出た・「ここをこう変えたい」——そういうときは、**開発担当に伝えるだけ**で大丈夫です。コードの修正も、修正後の再公開も自動で回ります(変更が本番に反映されると、サイトはひとりでに更新されます)。**運営が裏側のコードを触る必要はありません。**

大事なのは、抱え込まずにまず共有すること。何が起きても元に戻せますし、データが消えるような操作は運営の通常作業には含まれていません。

---

## 9. これから

いま動いているのは **Phase 1**——「状態の可視化」です。課題・成績・予定を、残して見えるようにするところまで。

土台が定着し、成績と分野タグ(例:数学>二次関数)が十分にたまったら、次は **Phase 2**——「この分野で連続して落としている → 次はここに戻ろう」という**学習提案**へと進みます。データは最初からその日のために設計してあるので、作り直しは要りません。

ただ、急ぐべきは Phase 2 ではなく、**まず"使われるアプリ"になること**。生徒が自然に触れる状態が四週間続いたら、次の扉を開けます。
